## 先說結論：loop 不是你想的那個迴圈

一句話要點：大家在講的「loop」不是程式裡 `for/while` 那種，而是一套「會自己被觸發、自己決定下一步、自己收尾」的自動化機制。

## 拆解綜合體：定時觸發 + Hook + 工具調用

一句話要點：現在最紅的 loop 是三件事的組合——定時觸發（什麼時候動）、Hook（被什麼事件勾起來動）、工具調用（動起來能做什麼）。

### 定時觸發：時間到了就自己跑

一句話要點：cron / 排程讓任務不需要你開口，到點就啟動。

### Hook：有事件就勾起來

一句話要點：某個動作完成或某個狀態改變時，自動觸發下一段流程，不用人盯著。

### 工具調用：真正去把事情做完

一句話要點：被觸發後，loop 會去調用既有的 Agent、Skill、MCP，把任務實際執行到完成。

## 為什麼這套組合現在這麼紅？

一句話要點：模型夠強只是基礎，能「自己動起來、不用人催」才是這波自動化的真正分水嶺。

## 它怎麼跟既有的 Agent / Skill / MCP 接起來

一句話要點：loop 不重造輪子，而是當調度層，把你已經寫好的 Skill、接好的 MCP、設好的 Agent 串成一條會自動跑的流水線。

## 鴨編的觀察：從「我下指令」到「它自己跑」

一句話要點：loop 代表互動模式的轉變——使用者從「每次手動下指令」變成「設定好條件，讓系統自己循環推進」。

## 小心：自動跑也要有煞車

一句話要點：定時 + 自動調用很強，但也要有邊界、驗證與停損機制（呼應 Harness 的角色），否則自動化會變成自動闖禍。
