# OpenClaw 新手安裝系列文章 - 結構設計指南

> 本文檔為 OpenClaw 新手安裝系列文章提供統一的結構模板與內容規劃指引

---

## 一、統一文章結構模板（適用於所有文章）

### 1.1 文章基本資訊區塊

```
┌─────────────────────────────────────────────────────────┐
│  文章標題                                                │
│  【系列編號】OpenClaw 新手入門 #X - XXXXXX              │
├─────────────────────────────────────────────────────────┤
│  目標讀者：完全沒經驗的新手                              │
│  預估閱讀時間：X 分鐘                                   │
│  難度等級：⭐☆☆☆☆ (入門)                               │
│  前置需求：[列出需要先閱讀的文章]                        │
└─────────────────────────────────────────────────────────┘
```

### 1.2 標準文章架構

```
┌─────────────────────────────────────────────────────────┐
│  1. 開場引言 (Introduction)                              │
│     └─ 為什麼要學這個？學完能做什麼？                   │
├─────────────────────────────────────────────────────────┤
│  2. 快速開始 (Quick Start) ⭐ 可選區塊                   │
│     └─ 3-5 步驟快速上手，適合有經驗的讀者               │
├─────────────────────────────────────────────────────────┤
│  3. 核心內容 (Main Content)                              │
│     └─ 依文章類型有不同的章節結構                        │
├─────────────────────────────────────────────────────────┤
│  4. 常見問題 (FAQ)                                       │
│     └─ 3-5 個新手最常問的問題                           │
├─────────────────────────────────────────────────────────┤
│  5. 下一步行動 (Next Steps)                              │
│     └─ 建議閱讀的下一篇文章                              │
├─────────────────────────────────────────────────────────┤
│  6. 相關資源 (Resources)                                 │
│     └─ 官方文件、推薦連結、延伸閱讀                     │
└─────────────────────────────────────────────────────────┘
```

### 1.3 統一格式規範

| 元素 | 格式規範 |
|------|----------|
| 標題 | H1 用於文章標題，H2 用於主要章節，H3 用於子章節 |
| 步驟編號 | 使用 `1.` `2.` `3.` 有序列表 |
| 重點提示 | 使用 `💡` 提示、`⚠️` 警告、`🎯` 重點 |
| 程式碼 | 使用程式碼區塊，標註語言類型 |
| 按鈕/選單 | 使用 **粗體** 標註，如：點擊 **設定** |

## 二、三種文章類型專屬結構

### 2.1 概念介紹型文章

**適用主題**：什麼是 LLM、什麼是 API、OpenClaw 是什麼

**結構模板**：

```
┌─────────────────────────────────────────────────────────┐
│  【開場引言】為什麼需要了解這個概念？                     │
│  ─────────────────────────────────────────────────────  │
│  • 用生活化的比喻開場（新手聽得懂）                      │
│  • 說明學習這個概念的必要性                              │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  【概念解釋】用白話說明核心概念                           │
│  ─────────────────────────────────────────────────────  │
│  • 避免專業術語，或術語出現時立即解釋                    │
│  • 使用類比/比喻幫助理解                                 │
│  • 搭配簡單圖示說明                                      │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  【運作原理】簡單說明「它是如何運作的」                   │
│  ─────────────────────────────────────────────────────  │
│  • 用流程圖或步驟說明                                    │
│  • 不需要太深入技術細節                                  │
│  • 重點是建立基本認知                                    │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  【實際應用】這個概念在 OpenClaw 中的應用                 │
│  ─────────────────────────────────────────────────────  │
│  • 連結到讀者即將要做的事                                │
│  • 舉 2-3 個具體例子                                     │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  【常見誤解】新手容易搞錯的地方                           │
│  ─────────────────────────────────────────────────────  │
│  • 澄清 2-3 個常見迷思                                   │
│  • 用對比表格呈現正確/錯誤觀念                           │
└─────────────────────────────────────────────────────────┘
```

**預估字數**：1,500 - 2,500 字

**內容重點**：
- 70% 白話解釋 + 30% 技術說明
- 每個新概念都要有類比
- 避免一次講太多概念

## 六、寫作風格指南

### 6.1 語氣與用詞

| ✅ 建議使用 | ❌ 避免使用 |
|------------|------------|
| 「點擊**設定**按鈕」 | 「進入設定介面」 |
| 「你會看到...」 | 「使用者將會看到...」 |
| 「如果發生錯誤...」 | 「在錯誤發生的情況下...」 |
| 「簡單來說...」 | 「簡言之...」 |
| 「不用擔心」 | 「無需擔憂」 |

### 6.2 句子長度建議

- 說明文字：每句不超過 20 個字
- 步驟說明：每步不超過 3 行
- 段落長度：每段不超過 5 行

### 6.3 新手友善原則

1. **每個專業術語第一次出現都要解釋**
2. **提供具體的例子而非抽象描述**
3. **預判可能出錯的地方並提前提醒**
4. **使用「你」而非「使用者」來增加親切感**

## 八、總結

本指南提供：

1. **統一結構模板** - 適用於所有文章的標準架構
2. **三種專屬結構** - 針對不同文章類型的最佳實踐
3. **雙軌呈現方式** - 快速開始 + 詳細說明的設計
4. **視覺輔助規劃** - 明確標註需要圖表的位置
5. **快速複製模板** - 可直接使用的 Markdown 模板

使用這些模板可以確保系列文章的一致性和專業性，同時保持對新手的友善度。
