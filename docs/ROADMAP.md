# 藍鴨 ROADMAP — vibe coding 課 × 限時能力包（試營運實驗）

> 建立：2026-07-10　｜　狀態：**假設待驗，非結論**　｜　時機：月底第一次試營運
> 這是一頁實驗規格，不是計畫書。目的：用最小成本驗證「限時開放能力包的 vibe coding 陪跑」值不值得做成訂閱。
> 現況背景見 `STATUS.md`（藍鴨進度僅在第一步）。

---

## 一句話決定
帶 1 人／一小群做 vibe coding、打造特定任務時，讓他們**限時接上藍鴨能力包**，用普通模型也能完成任務；課程/時間一到就關閉存取。順利再進第二階（持續買課＝持久存取）。

## 核心賣點（headline，該當標題打）
**「用普通、甚至免費的模型，也能把任務做完。」** 能力包讓廉價模型打出超規格表現——把成本邏輯翻過來，和藍鴨「別亂花錢、別 FOMO」立場一致。

## 商模假設（要被驗證，不是結論）
- **H1**　有人願意為「限時接上能力包的 vibe coding 陪跑」付費。
- **H2**　關閉存取後，有人願意續訂以保持存取＋持續更新（＝訂閱成立的前提）。
- **H3**　能力包能讓普通模型在**選定任務**上穩定完成（不是任何任務）。

## 要驗證的三個數字
1. **付費轉換率**：接觸／報名 → 實際付費。
2. **關閉後續訂率**：試營運結束、存取關閉後，願意續訂的比例。
3. **任務完成率**：普通模型 ＋ 能力包，在選定 demo 任務上的實際完成比例。

## 最小設計（Phase 1，別過度工程）
- **存取控制**：在 `memory-mcp` 前加「每梯次專屬、帶到期時間的 bearer token」，到期即失效。第一場人數少 → **手動發 token 就好**，別先蓋權限系統。
- **成本護欄**：能力包只給「方法/知識」，**不代呼叫付費 API**；模型由學員自備。避免一班猛打時帳單落到你頭上。
- **demo 任務**：從 `launchdock-lab` 挑 **2–3 個你確定會贏**的任務，範圍框窄。別開放式承諾「任何任務」，否則現場翻車、賣點反噬。
- **off-ramp（最容易被漏）**：**存取關閉的那一刻＝訂閱成交點**。當場明確給「續訂＝持續存取＋每月更新能力包」。ephemeral access 只有 wow、沒有 off-ramp＝零轉換。
- **護城河認知**：能收回「存取」，收不回「記憶」。別靠保密單一 prompt 建護城河——護城河是**持續更新的整包 ＋ 你的現場帶練**。讓能力包常更新，使舊複本快速過時。

## 分階
- **Phase 1（月底試營運）**：小班／1 對 1，限時能力包，跑選定任務，收上面三個數字。
- **Phase 1.5（平行，LINE 留存迴圈）**：報名/測驗 → 加 **LINE 官方帳號好友** → 推播新文/開課/更新 → 續訂。（登入 ≠ 能推播；要的是 OA 好友名單。）押注前先查 LINE OA 現行推播方案/報價。低成本先做的：文章「分享到 LINE」按鈕。
- **Phase 2（若 H1–H3 成立）**：持續買課＝訂閱制＝持久存取 ＋ 每月更新能力包 ＋ 現場帶練。

---

## 翻案條件（什麼證據會讓結論反轉）
- **任務完成率低** → 先修能力包／再縮任務範圍，**別急著開賣**。
- **單場滿意高但續訂率低** → 這不是訂閱，改成**一次性梯次賣斷**（cohort）。
- **方法一看即走、且一場就給滿價值** → 同上，賣斷，別做訂閱。
- **付費客群其實多為純非技術者、用不上能力包** → 付費核心改成「陪你做（done-with-you）」，能力包退居輔助。
