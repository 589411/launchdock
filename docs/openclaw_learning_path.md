## 🎯 为什么这样安排学习顺序？

### 【设计原则】

| 原则 | 说明 |
|------|------|
| **1. 从「为什么」到「怎么做」** | 先让新手理解 OpenClaw 是什么、能解决什么问题（#1-#3），再进入实际操作（#5 以后）。避免新手在不了解价值的情况下就放弃。 |
| **2. 先理解概念，再动手操作** | 文章 #2（LLM 基础）必须在 #5（选择 LLM 方案）之前，否则新手无法理解各种方案的差异。 |
| **3. 按需分叉，减少不必要的内容** | 文章 #6、#7、#8 是平行关系，根据用户选择的方案只读其中一篇，避免浪费时间在不相关的 API 获取教程上。 |
| **4. 必须有 API Key 才能安装** | 文章 #9/#10 的前置需求是已获得 API Key，因为安装后需要立即配置，否则无法验证安装是否成功。 |
| **5. 循序渐进的使用教学** | #11（界面导览）→ #12（连接 LLM）→ #13（使用 Skill），每一步都建立在前一步的基础上，没有知识断层。 |
| **6. 支線内容不阻塞主線进度** | 文章 #4、#8、#14 是支線，跳过它们不会影响主線学习，可以在需要时随时查阅。 |
