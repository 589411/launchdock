# OpenClaw 新手技術概念指南

> 完全零經驗也能懂的技術名詞解釋

---

## 1. LLM（大型語言模型）

| 項目 | 內容 |
|------|------|
| **術語** | LLM（Large Language Model / 大型語言模型） |
| **一句話解釋** | 讀過海量書籍、能理解和生成人類語言的 AI 大腦。 |
| **生活化比喻** | 🧠 想像一個讀過全世界圖書館的超級學霸，你問什麼它都能回答，還能幫你寫文章、翻譯、解釋概念。 |
| **詳細說明** | LLM 是一種 AI 程式，它透過學習網路上大量的文字資料，學會了語言的規則和知識。它就像一個超級助手，能聽懂你的問題並給出合理的回答。常見的 LLM 包括 ChatGPT、Claude、Gemini 等。 |
| **與 OpenClaw 的關聯** | OpenClaw 的核心就是讓你能輕鬆使用各種 LLM，不需要懂程式也能和 AI 對話。 |
| **需要圖表嗎？** | ✅ 建議 - 可用「大腦 + 書本」的簡單圖示 |

---

## 2. CoT（Chain of Thought / 思維鏈）

| 項目 | 內容 |
|------|------|
| **術語** | CoT（Chain of Thought / 思維鏈） |
| **一句話解釋** | 讓 AI 像人類一樣「一步步思考」再回答的技術。 |
| **生活化比喻** | 📝 就像解數學題時寫「計算過程」，不是直接寫答案，而是把思考的步驟一條條列出來，這樣答案更正確。 |
| **詳細說明** | 傳統 AI 可能直接給答案，但 CoT 會先說「首先...然後...最後...」。這種「自言自語」的思考過程讓 AI 處理複雜問題時更準確，就像我們寫草稿比直接寫正式答案更不容易出錯。 |
| **與 OpenClaw 的關聯** | OpenClaw 的某些進階功能會自動啟用 CoT，讓 AI 幫你分析複雜問題時更可靠。 |
| **需要圖表嗎？** | ✅ 建議 - 可用「步驟箭頭流程圖」展示思考過程 |

---

## 3. MCP（Model Context Protocol）

| 項目 | 內容 |
|------|------|
| **術語** | MCP（Model Context Protocol / 模型上下文協議） |
| **一句話解釋** | AI 和外部工具之間的「通用翻譯官」標準。 |
| **生活化比喻** | 🔌 就像 USB-C 充電線，不管什麼品牌的充電器都能插。MCP 讓不同的 AI 和工具能互相溝通，不需要為每個組合單獨設定。 |
| **詳細說明** | MCP 是一種開放標準，讓 AI 能輕鬆連接各種外部服務（如搜尋引擎、資料庫、檔案系統）。以前每個 AI 和工具組合都要單獨寫程式連接，現在只要支援 MCP，就能「一插即用」。 |
| **與 OpenClaw 的關聯** | OpenClaw 支援 MCP，讓你能輕鬆擴展 AI 的能力，連接更多實用工具。 |
| **需要圖表嗎？** | ✅ 強烈建議 - 可用「插頭插座」或「翻譯官」圖示 |

---

## 4. Skill（技能）

| 項目 | 內容 |
|------|------|
| **術語** | Skill（技能） |
| **一句話解釋** | AI 學會的某項特定能力或專長。 |
| **生活化比喻** | 🎯 就像人學會「開車」、「煮飯」、「說英文」一樣，AI 的 Skill 就是它能做的特定事情，比如「搜尋網路」、「讀取檔案」、「寫程式」。 |
| **詳細說明** | Skill 是 AI 系統中的功能模組，每個 Skill 專注做一件事。你可以把多個 Skill 組合起來，讓 AI 完成更複雜的任務。就像組合樂高積木，每塊積木（Skill）都有特定功能。 |
| **與 OpenClaw 的關聯** | OpenClaw 提供各種內建 Skill，新手可以直接使用，不需要自己開發。 |
| **需要圖表嗎？** | ✅ 建議 - 可用「工具箱」或「樂高積木」圖示 |

---

## 5. Agent（智能代理）

| 項目 | 內容 |
|------|------|
| **術語** | Agent（智能代理 / AI 代理） |
| **一句話解釋** | 能自主規劃步驟、使用工具完成任務的 AI 助手。 |
| **生活化比喻** | 🤖 就像一個貼身助理，你說「幫我安排東京之旅」，它會自己查機票、找飯店、規劃行程，不用你一步步指揮。 |
| **詳細說明** | 普通 AI 只會回答問題，但 Agent 能「動手做事」。它會分析任務、規劃步驟、呼叫各種工具（搜尋、計算、發信等），直到完成目標。你給它目標，它自己想辦法達成。 |
| **與 OpenClaw 的關聯** | OpenClaw 可以設定為 Agent 模式，讓 AI 自動幫你完成多步驟的複雜任務。 |
| **需要圖表嗎？** | ✅ 強烈建議 - 可用「機器人助手」或「循環工作流程圖」 |

---

## 6. 集群（Cluster）

| 項目 | 內容 |
|------|------|
| **術語** | Cluster（集群） |
| **一句話解釋** | 多台電腦合作工作，像一個團隊共同完成任務。 |
| **生活化比喻** | 👥 就像搬重物時，一個人搬不動，找一群朋友一起搬。集群就是讓多台電腦「團隊合作」，處理單台電腦做不來的大任務。 |
| **詳細說明** | 集群將多台電腦連接起來，讓它們像一台超級電腦一樣工作。這樣可以提高處理速度、增加穩定性（一台壞了其他還能繼續），還能處理更大的資料量。 |
| **與 OpenClaw 的關聯** | 進階用戶可以設定 OpenClaw 使用集群模式，讓 AI 處理能力更強大、回應更快。 |
| **需要圖表嗎？** | ✅ 建議 - 可用「多台電腦連接」或「團隊合作」圖示 |

## 7. API（應用程式介面）

| 項目 | 內容 |
|------|------|
| **術語** | API（Application Programming Interface / 應用程式介面） |
| **一句話解釋** | 不同程式之間「對話」的標準方式。 |
| **生活化比喻** | 🍽️ 就像餐廳的服務生，你（A程式）不需要進廚房，只要跟服務生點菜，服務生會幫你把需求傳給廚房（B程式），再把做好的菜端給你。 |
| **詳細說明** | API 是一組規則，讓不同的軟體能互相溝通。A程式想使用 B程式的功能時，透過 API 發送請求，B程式處理後回傳結果。這樣開發者不需要知道 B程式的內部運作，就能使用它的功能。 |
| **與 OpenClaw 的關聯** | OpenClaw 透過 API 連接各種 AI 服務和工具，新手設定時可能需要輸入 API 金鑰。 |
| **需要圖表嗎？** | ✅ 強烈建議 - 可用「餐廳點餐流程」或「請求-回應箭頭圖」 |

## 8. Token（令牌）

| 項目 | 內容 |
|------|------|
| **術語** | Token（令牌） |
| **一句話解釋** | AI 處理文字時的最小單位，也是計費的基礎。 |
| **生活化比喻** | 🧩 就像樂高積木的基本單塊，AI 把文字拆成一個個 Token 來理解和計算。英文約 4 個字母 = 1 Token，中文約 1 個字 = 1 Token。 |
| **詳細說明** | Token 是 AI 處理語言的基本單位。當你發送問題給 AI，它會先把文字轉成 Token 序列來理解；回答時也是一個個 Token 生成出來的。大多數 AI 服務按 Token 數量收費，就像按字數計費一樣。 |
| **與 OpenClaw 的關聯** | 了解 Token 能幫你估算使用 OpenClaw 的成本，長對話會消耗更多 Token。 |
| **需要圖表嗎？** | ✅ 建議 - 可用「文字被拆分成小塊」的示意圖 |
