## 🎯 为什么这样安排学习顺序？

### 【设计原则】

| 原则 | 说明 |
|------|------|
| **1. 从「为什么」到「怎么做」** | 先让新手理解 OpenClaw 是什么、能解决什么问题（#1-#3），再进入实际操作（#5 以后）。避免新手在不了解价值的情况下就放弃。 |
| **2. 先理解概念，再动手操作** | 文章 #2（LLM 基础）必须在 #5（选择 LLM 方案）之前，否则新手无法理解各种方案的差异。 |
| **3. 按需分叉，减少不必要的内容** | 文章 #6、#7、#8 是平行关系，根据用户选择的方案只读其中一篇，避免浪费时间在不相关的 API 获取教程上。 |
| **4. 必须有 API Key 才能安装** | 文章 #9/#10 的前置需求是已获得 API Key，因为安装后需要立即配置，否则无法验证安装是否成功。 |
| **5. 循序渐进的使用教学** | #11（界面导览）→ #12（连接 LLM）→ #13（使用 Skill），每一步都建立在前一步的基础上，没有知识断层。 |
| **6. 支線内容不阻塞主線进度** | 文章 #4、#8、#14 是支線，跳过它们不会影响主線学习，可以在需要时随时查阅。 |

# 二、技術概念簡化指南

# OpenClaw 新手技術概念指南

> 完全零經驗也能懂的技術名詞解釋

---

## 1. LLM（大型語言模型）

| 項目 | 內容 |
|------|------|
| **術語** | LLM（Large Language Model / 大型語言模型） |
| **一句話解釋** | 讀過海量書籍、能理解和生成人類語言的 AI 大腦。 |
| **生活化比喻** | 🧠 想像一個讀過全世界圖書館的超級學霸，你問什麼它都能回答，還能幫你寫文章、翻譯、解釋概念。 |
| **詳細說明** | LLM 是一種 AI 程式，它透過學習網路上大量的文字資料，學會了語言的規則和知識。它就像一個超級助手，能聽懂你的問題並給出合理的回答。常見的 LLM 包括 ChatGPT、Claude、Gemini 等。 |
| **與 OpenClaw 的關聯** | OpenClaw 的核心就是讓你能輕鬆使用各種 LLM，不需要懂程式也能和 AI 對話。 |
| **需要圖表嗎？** | ✅ 建議 - 可用「大腦 + 書本」的簡單圖示 |

---

## 2. CoT（Chain of Thought / 思維鏈）

| 項目 | 內容 |
|------|------|
| **術語** | CoT（Chain of Thought / 思維鏈） |
| **一句話解釋** | 讓 AI 像人類一樣「一步步思考」再回答的技術。 |
| **生活化比喻** | 📝 就像解數學題時寫「計算過程」，不是直接寫答案，而是把思考的步驟一條條列出來，這樣答案更正確。 |
| **詳細說明** | 傳統 AI 可能直接給答案，但 CoT 會先說「首先...然後...最後...」。這種「自言自語」的思考過程讓 AI 處理複雜問題時更準確，就像我們寫草稿比直接寫正式答案更不容易出錯。 |
| **與 OpenClaw 的關聯** | OpenClaw 的某些進階功能會自動啟用 CoT，讓 AI 幫你分析複雜問題時更可靠。 |
| **需要圖表嗎？** | ✅ 建議 - 可用「步驟箭頭流程圖」展示思考過程 |

---

## 3. MCP（Model Context Protocol）

| 項目 | 內容 |
|------|------|
| **術語** | MCP（Model Context Protocol / 模型上下文協議） |
| **一句話解釋** | AI 和外部工具之間的「通用翻譯官」標準。 |
| **生活化比喻** | 🔌 就像 USB-C 充電線，不管什麼品牌的充電器都能插。MCP 讓不同的 AI 和工具能互相溝通，不需要為每個組合單獨設定。 |
| **詳細說明** | MCP 是一種開放標準，讓 AI 能輕鬆連接各種外部服務（如搜尋引擎、資料庫、檔案系統）。以前每個 AI 和工具組合都要單獨寫程式連接，現在只要支援 MCP，就能「一插即用」。 |
| **與 OpenClaw 的關聯** | OpenClaw 支援 MCP，讓你能輕鬆擴展 AI 的能力，連接更多實用工具。 |
| **需要圖表嗎？** | ✅ 強烈建議 - 可用「插頭插座」或「翻譯官」圖示 |

---

## 4. Skill（技能）

| 項目 | 內容 |
|------|------|
| **術語** | Skill（技能） |
| **一句話解釋** | AI 學會的某項特定能力或專長。 |
| **生活化比喻** | 🎯 就像人學會「開車」、「煮飯」、「說英文」一樣，AI 的 Skill 就是它能做的特定事情，比如「搜尋網路」、「讀取檔案」、「寫程式」。 |
| **詳細說明** | Skill 是 AI 系統中的功能模組，每個 Skill 專注做一件事。你可以把多個 Skill 組合起來，讓 AI 完成更複雜的任務。就像組合樂高積木，每塊積木（Skill）都有特定功能。 |

### 步驟 1：安裝 Node.js（執行環境）

| 屬性 | 內容 |
|-----|------|
| **預估時間** | 10-15 分鐘 |
| **難度等級** | ⭐⭐ 中等 |

#### 操作說明 - macOS

1. **開啟終端機**
   - 按 `Cmd + 空白鍵` 開啟 Spotlight
   - 輸入 `Terminal` 並開啟

2. **安裝 Homebrew（套件管理器）**
   ```bash
   /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
   ```
   - 依照提示輸入密碼
   - 等待安裝完成（約 5-10 分鐘）

3. **安裝 Node.js**
   ```bash
   brew install node@22
   ```

4. **驗證安裝**
   ```bash
   node --version
   ```

#### 操作說明 - Linux (Ubuntu/Debian)

1. **開啟終端機**
   - 按 `Ctrl + Alt + T`

2. **安裝 Node.js**
   ```bash
   curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
   sudo apt-get install -y nodejs
   ```

3. **驗證安裝**
   ```bash
   node --version
   ```

#### 操作說明 - Windows（使用 WSL2 - 推薦）

1. **以系統管理員身分開啟 PowerShell**
   - 右鍵點選 PowerShell → 以系統管理員身分執行

2. **安裝 WSL2**
   ```powershell
   wsl --install
   ```
   - 安裝完成後**重新啟動電腦**

3. **設定 Ubuntu**
   - 重啟後，Ubuntu 會自動開啟
   - 設定使用者名稱和密碼

4. **在 Ubuntu 中安裝 Node.js**
   ```bash
   curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
   sudo apt-get install -y nodejs
   ```

#### 驗證方法
```bash
node --version
npm --version
```
✅ 應顯示版本號：
- Node.js: v22.x.x 或更高
- npm: 10.x.x 或更高

#### 常見錯誤與解決方案

| 錯誤訊息 | 原因 | 解決方案 |
|---------|------|---------|
| `command not found: node` | 安裝未完成或路徑未設定 | 重新安裝，或執行 `exec bash` 重新載入 |
| `EACCES: permission denied` | 權限不足 | 在命令前加 `sudo` |
| `brew: command not found` | Homebrew 未安裝 | 重新執行 Homebrew 安裝指令 |
| WSL 安裝卡住 | 虛擬化未啟用 | 在 BIOS 啟用 Intel VT-x/AMD-V |

### 步驟 4：設定訊息平台（以 Telegram 為例）

| 屬性 | 內容 |
|-----|------|
| **預估時間** | 10 分鐘 |
| **難度等級** | ⭐⭐ 中等 |

#### 操作說明

1. **在精靈中選擇 Telegram**
   - 使用方向鍵選擇 "Telegram"
   - 按 Enter

2. **開啟 Telegram 應用程式**
   - 手機或電腦版都可以

3. **搜尋 @BotFather**
   - 在搜尋框輸入 `@BotFather`
   - 點擊開始對話

4. **建立新機器人**
   - 發送指令： `/newbot`
   - 輸入機器人名稱（如：MyOpenClawBot）

5. **複製 API Token**
   - BotFather 會回覆一個 token，格式如：`123456789:ABCdefGHIjklMNOpqrSTUvwxyz`
   - **複製這個 token**（點擊即可複製）

6. **貼上 Token 到精靈**
   - 回到終端機
   - 貼上 token
   - 按 Enter

7. **配對確認**
   - 在 Telegram 中對你的機器人發送任意訊息
   - 精靈會顯示一組配對碼
   - 在 Telegram 中確認配對

#### 驗證方法

1. **測試機器人回應**
   - 在 Telegram 對你的機器人發送：`Hello`
   - 等待 5-10 秒

2. **確認回應**
   - ✅ 機器人應回覆問候訊息
   - ✅ 表示連線成功

#### 常見錯誤與解決方案

| 錯誤訊息 | 原因 | 解決方案 |
|---------|------|---------|
| `Invalid token` | Token 複製不完整 | 重新向 BotFather 取得 token |
| `Bot not responding` | Gateway 未啟動 | 執行 `openclaw gateway restart` |
| `Access not configured` | 未配對 | 執行 `openclaw pairing approve telegram <CODE>` |
| `Bot is already in use` | Token 被其他服務使用 | 建立新的機器人 |

| 屬性 | 內容 |

| **預估時間** | 5 分鐘 |
| **難度等級** | ⭐⭐ 中等 |

#### 操作說明

1. **編輯設定檔**
   ```bash
   nano ~/.openclaw/openclaw.json
   ```

2. **修改 Gateway 綁定設定**
   ```json
   {
     "gateway": {
       "host": "127.0.0.1",
       "port": 18789,
       "auth_token": "你的強密碼"
     }
   }
   ```
   - 將 `host` 從 `0.0.0.0` 改為 `127.0.0.1`（僅本地訪問）
   - 設定強密碼作為 `auth_token`

3. **啟用執行確認模式**
   ```json
   {
     "exec_approval": true
   }
   ```
   這會在執行危險指令前要求確認

4. **設定 Allowlist（Telegram）**
   ```json
   {
     "telegram": {
       "allowlist": ["你的 Telegram 使用者名稱"]
     }
   }
   ```

5. **重啟 Gateway**
   ```bash
   openclaw gateway restart
   ```

#### 驗證方法
- [ ] 從其他設備無法直接訪問 `http://你的IP:18789`
- [ ] 訪問 `http://localhost:18789` 需要輸入 token

#### 常見錯誤與解決方案

| 錯誤訊息 | 原因 | 解決方案 |
|---------|------|---------|
| `JSON parse error` | 設定檔格式錯誤 | 檢查逗號和括號 |
| `Config not loaded` | 設定檔路徑錯誤 | 確認檔案位置正確 |

### 步驟 9：設定自動啟動（可選）

| 屬性 | 內容 |
|-----|------|
| **預估時間** | 3 分鐘 |
| **難度等級** | ⭐⭐ 中等 |

#### 操作說明 - macOS

1. **建立 LaunchAgent**
   ```bash
   nano ~/Library/LaunchAgents/com.openclaw.gateway.plist
   ```

2. **加入以下內容**
   ```xml
   <?xml version="1.0" encoding="UTF-8"?>
   <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN">
   <plist version="1.0">
   <dict>
     <key>Label</key>
     <string>com.openclaw.gateway</string>
     <key>ProgramArguments</key>
     <array>
       <string>/usr/local/bin/openclaw</string>
       <string>gateway</string>
       <string>start</string>
     </array>
     <key>RunAtLoad</key>
     <true/>
     <key>KeepAlive</key>
     <true/>
   </dict>
   </plist>
   ```

3. **載入服務**
   ```bash
   launchctl load ~/Library/LaunchAgents/com.openclaw.gateway.plist
   ```

#### 操作說明 - Linux

如果使用 `--install-daemon`，已自動設定 systemd 服務：
```bash
# 檢查服務狀態
sudo systemctl status openclaw

# 啟用開機自動啟動
sudo systemctl enable openclaw
```

- [ ] 重啟電腦後，執行 `openclaw gateway status` 顯示 "running"

## 五、常見問題快速排解

### 5.1 安裝問題

### 5.2 運行問題

| 問題 | 快速解決 |

| Telegram 無回應 | 確認 token 正確，Gateway 運行中 |
| 回應很慢 | 檢查網路，或換用較輕量模型 |

### 5.3 安全問題

| 問題 | 快速解決 |
|-----|---------|
| Gateway 暴露在外網 | 修改 `host` 為 `127.0.0.1` |
| 忘記 auth token | 編輯 `~/.openclaw/openclaw.json` 重設 |
| 收到陌生人的訊息 | 設定 Telegram allowlist |

```bash
openclaw status

npm update -g openclaw
```

## 六、寫作風格指南

### 6.1 語氣與用詞

| ✅ 建議使用 | ❌ 避免使用 |
|------------|------------|
| 「點擊**設定**按鈕」 | 「進入設定介面」 |
| 「你會看到...」 | 「使用者將會看到...」 |
| 「如果發生錯誤...」 | 「在錯誤發生的情況下...」 |
| 「簡單來說...」 | 「簡言之...」 |
| 「不用擔心」 | 「無需擔憂」 |

### 6.2 句子長度建議

- 說明文字：每句不超過 20 個字
- 步驟說明：每步不超過 3 行
- 段落長度：每段不超過 5 行

### 6.3 新手友善原則

1. **每個專業術語第一次出現都要解釋**
2. **提供具體的例子而非抽象描述**
3. **預判可能出錯的地方並提前提醒**
4. **使用「你」而非「使用者」來增加親切感**

## 三、FAQ 问题列表（20个常见问题）

### 基础概念类

**Q1: OpenClaw 是免费的吗？**
> OpenClaw 本身是开源免费的，但你需要付费给 AI 服务商（如 OpenAI、Anthropic）。每次对话约 $0.01-0.1 美元，取决于你使用的 AI 模型和对话长度。

**Q2: 什么是 API Key？我需要申请几个？**
> API Key 就像是一把钥匙，让 OpenClaw 能够呼叫 AI 服务。你只需要申请一个（推荐从 OpenAI 或 Anthropic 开始），把它设定到 OpenClaw 里即可。

**Q3: 我不懂程序，可以安装 OpenClaw 吗？**
> 可以！本系列文章是专为完全新手设计的，只要按照步骤一步步做，不需要写程序也能成功安装。

**Q4: OpenClaw 会偷看我的资料吗？**
> OpenClaw 是运行在你自己电脑上的，你的对话记录和资料都储存在本地，不会自动上传到云端（除非你使用云端 AI 服务，那时只有对话内容会传给 AI 服务商）。

**Q5: 我的电脑很旧，可以跑 OpenClaw 吗？**
> 最低需求：4GB RAM、10GB 硬盘空间、Windows 10/macOS 11/Ubuntu 20.04 以上。如果只有 4GB RAM，回应会比较慢，建议至少 8GB。

---

### 安装过程类

**Q6: 安装时出现「command not found」怎么办？**
> 这表示系统找不到这个指令。可能原因：① 打错字了，请重新复制贴上；② 需要先安装 Node.js，请参考第2篇的「准备环境」章节。

**Q7: 安装到一半卡住很久，是正常的吗？**
> 如果是下载阶段，可能需要 5-10 分钟（取决于网速）。如果超过 20 分钟没动静，可以按 `Ctrl+C` 取消后重试。

**Q8: 我可以用 Windows 内建的 PowerShell 吗？**
> 可以，但建议使用 Windows Terminal 或 Git Bash，显示效果会更好。PowerShell 也能用，只是有些指令的写法会稍微不同。

**Q9: 安装完成后，我怎么知道成功了？**
> 在终端输入 `openclaw --version`，如果看到版本号码（如 `2026.2.6`），就表示安装成功了！

**Q10: 我可以同时安装多个版本的 OpenClaw 吗？**
> 不建议。如果之前安装过旧版本，建议先完整移除后再安装新版本，避免冲突。

---

### 设定与使用类

**Q11: API Key 要贴到哪里？**
> 贴到 OpenClaw 的设定档里。档案位置是 `~/.openclaw/openclaw.json`（Windows 是 `C:\Users\你的用户名\.openclaw\openclaw.json`）。用记事本打开，找到 `api_key` 栏位贴上即可。

**Q12: 我贴好 API Key 了，但 OpenClaw 说「认证失败」？**
> 可能原因：① Key 复制不完整（要完整复制包括 `sk-` 开头）；② Key 还没启用（有些服务商需要等几分钟）；③ 余额用完了。请检查这三点。

**Q13: OpenClaw 回应好慢，是正常的吗？**
> 第一次回应通常需要 10-30 秒，因为 AI 在「思考」。如果每次都超过 1 分钟，可能是：① 网络问题；② 使用了太复杂的模型；③ 电脑规格不足。

**Q14: 我可以跟 OpenClaw 用中文对话吗？**
> 可以！OpenClaw 支持中文，直接用中文打字即可。AI 模型也懂中文，会用中文回覆你。

**Q15: 对话记录存在哪里？我可以删除吗？**
> 存在 `~/.openclaw/` 资料夹里的 SQLite 数据库中。可以删除，但删除后 OpenClaw 就会「忘记」之前的对话内容。

---

### 费用与安全类

**Q16: 怎么控制 API 费用不要爆掉？**
> 方法：① 在 AI 服务商网站设定「用量上限」；② 使用较便宜的模型（如 GPT-3.5 而不是 GPT-4）；③ 定期查看用量仪表板。

**Q17: 我不小心把 API Key 公开了，怎么办？**
> 立即到 AI 服务商网站「撤销」这个 Key，然后重新申请一个新的。公开的 Key 可能被他人盗用，产生费用。

**Q18: OpenClaw 会不会自己一直跑，产生费用？**
> 预设情况下，OpenClaw 只会在你发送讯息时才呼叫 API。但如果有设定「自动任务」，就可能在背景执行。建议新手先关闭自动任务功能。

**Q19: 我的 API 费用异常高，可能是什么原因？**
> 可能原因：① 使用了太贵的模型；② 对话太长（Token 数太多）；③ 有自动任务在背景执行；④ Key 被盗用了。请检查用量明细。

---

### 故障排除类

**Q20: OpenClaw 突然不回应了，怎么办？**
> 排查步骤：
> 1. 检查 OpenClaw 是否还在运行（看终端窗口是否还在）
> 2. 检查网络连接是否正常
> 3. 检查 API Key 是否还有效、余额是否足够
> 4. 尝试重新启动 OpenClaw
> 5. 如果还是不行，查看错误日志（在 `~/.openclaw/logs/`）

| 鼓励用词 | 使用场景 |
|----------|----------|
| 「别担心」「没关系」 | 介绍可能出错的地方时 |

3. **庆祝小胜利**：每个完成检核清单都值得一句「恭喜你完成了！」
4. **不假设知识**：每个专业术语第一次出现时都要解释
5. **提供退路**：每个步骤都要说明「如果这一步做不到，可以先...」
